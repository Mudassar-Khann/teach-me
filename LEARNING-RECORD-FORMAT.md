# Learning Record Format

Learning records live in `./learning-records/` and use sequential numbering: `0001-slug.md`[cite: 3]. Create the directory lazily — only when the first record is written[cite: 3].

They capture non-obvious lessons, corrected misconceptions, and establish the zone of proximal development[cite: 3].

## Template

---
# (Optional) Metadata for spaced-repetition and lifecycle tracking.
status: active | superseded by LR-NNNN
confidence-score: 1-5
next-review-date: YYYY-MM-DD
---

# {Short title of what was learned or established}

{1-3 sentences: what was learned (or what prior knowledge was established), and why it matters for future sessions.}

## The Shift (Optional)
- **Previous mental model:** {What the user thought before}
- **The correction:** {Why it failed and what the correct model is}

## Evidence of Mastery
{How the user demonstrated the understanding — e.g., a question answered, an exercise completed, prior experience cited.}

## When to write a learning record
Write one when any of these is true[cite: 3]:
1. **The user demonstrated genuine understanding of something non-trivial**[cite: 3].
2. **The user disclosed prior knowledge**[cite: 3].
3. **A misconception was corrected**[cite: 3].
4. **The mission shifted in response to learning**[cite: 3].
